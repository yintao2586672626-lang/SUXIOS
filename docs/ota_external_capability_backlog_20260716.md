## 推荐实施顺序

1. `P0-01 → P0-02`：先让每条诊断都能以证据进入任务闭环。
2. `P0-03`：让SOP能被岗位与场景找到，并生成任务草稿。
3. `P0-04 → P1-01 → P1-04`：形成竞争趋势、异常、任务和复盘。
4. `P1-02 / P1-03 / P1-05`：补深钻、节假日节奏和工作表。
5. 核心闭环稳定后再做P2。

## 每次复核要回答的四个问题

1. 本周最值得完成的一个板块是什么，为什么？
2. 它依赖的真实字段、页面和保存回读链是否已经存在？
3. 还有哪些数据只能称OTA渠道信号，不能称酒店经营事实？
4. 完成后用什么最小证据验收，何时停止继续扩展？
