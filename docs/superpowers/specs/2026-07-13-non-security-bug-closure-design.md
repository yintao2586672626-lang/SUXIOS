# 非安全问题闭环修复设计（方案 A）

## 目标

在不重构整体架构、不新增业务功能的前提下，修复复盘中仍可在当前 HEAD 复现的 17 项非安全问题。保持携程/美团各自采集实现，只收紧状态、持久化、回读、日期、展示和测试契约。

## 已确认范围

### OTA 数据真实性

- 携程 `blocked`、`display_only`、`no_saved` 不再被前端或通知当成“已入库成功”。传输成功与持久化成功分开表达。
- 携程核心指标只接受真实数值；`null`、`--`、空字符串和非数字保持缺失，真实 `0` 保持为零。
- 美团直接保存路径必须满足 `saved_count > 0` 且数据库回读一致；零保存和回读不一致返回明确失败阶段。
- Profile 自动采集使用既有 0–9 天复用规则；10 天到期、未验证或明确登录失效不得显示可复用。当天 P0/数据质量门禁仍要求同日证明。
- 旧 OTA 凭证按“酒店 + 平台”分组迁移，只保留一个确定的 `ready` 凭证；其余旧配置显式标记为已被替代，配置元数据与 vault 状态一致。

### 分析与展示真实性

- 删除 CRC32 与 1.15–1.35 系数生成的“全渠道 AI 预计总间夜数”。没有真实来源字段时保持缺失，不制造预测。
- MacroSignal 的数据库读取失败显式返回 `read_failed`；收益、需求、价格和渠道卡片分别判断自己的样本，缺失值不显示为 0 或“预测可用”。
- 快速切换酒店时，旧请求不得覆盖新酒店；酒店身份不再仅凭“本店”或首行回退确定。
- 页面和导出使用空值判断而非真值判断，真实零值显示为 `0`。
- 双 OTA 连接状态从真实数据源/就绪状态生成，默认空状态不再硬编码已连接。
- 开业 AI 覆盖率先基于全量项目计算，再截取最多 6 条用于展示。
- 草稿恢复时 checkbox 同时支持 `false -> true` 与 `true -> false`。
- 转化率差使用“个百分点”，不再标成 `%`。

### 时间、HTTP 与工程门禁

- 所有业务默认日期按本地日历日生成；上海凌晨不再落到 UTC 前一天。时间戳仍保留 ISO UTC 语义。
- Conditional GET 遵守优先级：存在 `If-None-Match` 时忽略 `If-Modified-Since`。
- PowerShell pre-commit 对每个原生命令检查 `$LASTEXITCODE`，失败立即退出且不打印通过。
- CI 增加统一 Node 合同测试入口，执行全部 `tests/automation/*.test.mjs`；浏览器 `.spec.js` 不混入该步骤。

## 当前 HEAD 已解决项

复盘问题 11（`verify_e2e_contracts.mjs` 仍只检查 `public/index.html`）已由后续前端拆分提交解决。当前验证器读取 `public/index.html`、`resources/frontend/app-template.html` 和 `public/app-main.js`，`npm run verify:e2e-contracts` 已通过 2219 项检查，本次不重复修改。

## 非目标

- 不处理安全、合规、发布治理和真实 OTA 登录。
- 不改导航、视觉体系、数据库结构或采集技术路线。
- 不将 OTA 渠道数据扩大为全酒店经营事实。
- 不引入新的公共语义层或整体重构。

## 验收

- 每项生产代码修改前有能复现原问题的失败测试，修改后同一测试转绿。
- OTA 成功必须能区分获取、保存和回读阶段；缺失、零值和失败状态保持真实。
- 前端源文件、生成产物和契约守卫一致。
- 聚焦测试、PHPUnit、全部 Node 合同测试、P0 guards、非安全复盘验证和路由覆盖给出新鲜结果。
- 本地服务可启动时刷新相关页面；无法使用真实 OTA/凭证时明确标记该部分未验证，不伪造线上结果。
