# 宿析OS云端独立试用副本设计

## 目标

在现有腾讯云轻量服务器上运行一套可通过公网访问的宿析OS试用副本。云端与本地在数据库、配置、用户、酒店数据、OTA凭证和定时任务上完全独立；本地继续作为主系统，云端不做双写、自动同步或主备切换。

## 当前事实

- Ubuntu、Nginx、PHP 8.3、Composer 与 MariaDB 已安装并运行。
- 干净测试发布包已上传到独立版本目录，尚未配置 `.env`、业务数据库或 Nginx 入口。
- 初始化 SQL 含旧演示用户、酒店和业务记录，不能直接作为云端真实数据导入。

## 架构

请求链路为：公网 HTTPS -> Nginx -> PHP-FPM -> 宿析OS -> 本机 MariaDB。

- Nginx 仅暴露 `public/`，应用通过 `current` 软链接指向版本目录，便于回退。
- MariaDB 只监听 `127.0.0.1`，不开放公网 3306。
- 云端配置保存在 Git 与发布包之外，由 Web 用户只读访问。
- 本地系统不连接云端数据库，云端也不连接本地数据库。

## 数据边界

- 新建云端独立数据库与最小权限应用账号。
- 只导入表结构、必要角色、字段配置、行政区划和知识库种子。
- 不导入旧用户、旧酒店、业务日报、OTA数据、操作日志、Cookie、Profile或API密钥。
- 新建唯一的云端管理员账号并生成随机临时密码。
- OTA和AI功能初始保持未配置，页面必须如实显示缺少凭证或数据，不使用本地数据兜底。

## 访问与安全

- 无域名阶段使用公网IP与自签名 HTTPS；浏览器会提示证书非受信任，但传输内容加密。
- HTTP仅用于跳转到HTTPS；若腾讯云防火墙尚未放通443，则由用户在控制台添加TCP 443允许规则。
- `.env`、SQL、备份和运行目录禁止由Nginx直接访问。
- 之前在聊天中出现过的云API密钥不得进入服务器配置或发布包。

## 备份

- 每日凌晨执行一次 MariaDB 逻辑备份，保留最近7天。
- 备份目录仅root可读，输出gzip压缩文件、SHA-256校验文件和简短日志。
- 每次备份后执行压缩完整性检查；失败保留错误状态，不删除上一份成功备份。
- 同机备份只能应对误操作，不能替代异机容灾；试用阶段接受该边界。

## 验证与停止条件

部署完成前必须验证：

1. Nginx与PHP-FPM配置检查通过。
2. 公网HTTPS首页和 `/api/health` 返回成功。
3. 新管理员可以登录。
4. 云端数据库可写入并读回一条明确标记为云端试用的数据。
5. 本地数据库未被访问或修改。
6. 手动运行一次备份，gzip与SHA-256校验通过。
7. OTA与AI凭证保持未配置，且没有自动采集任务运行。

达到以上条件后停止，不迁移本地业务数据，不配置正式域名，不扩展为生产高可用架构。
