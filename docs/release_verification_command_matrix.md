# Release Verification Command Matrix

Updated: 2026-07-08

Scope: `@github`, `@openai-developers`, `@codex-security`, `@figma`, `@canva`

Purpose: keep each release issue tied to one isolated acceptance command before running the global release gate. Closed issues remain in the matrix for final-head reruns.

## Command Matrix

## Global Gates

Run these only after the isolated command for each closed blocker passes.

| Gate | Command | Expected current status | Purpose |
|---|---|---|---|
| Evidence bundle preflight | `npm run review:release-evidence` | Fails while design handoff or OTA credential rotation evidence is missing. | One read-only preflight for the controlled evidence directory plus repo-fixed release evidence paths. By default it writes `release-evidence-result.json` to `RELEASE_EVIDENCE_DIR` / `../release-evidence-temp` outside the repository and rejects repo-internal result files. |
| Evidence draft preparation | `npm run prepare:release-evidence-drafts` | Writes draft-only templates outside the repository. | Creates `drafts/design_handoff_manifest.draft.json`, `drafts/ota_credential_rotation_attestation.draft.json`, and a draft index under `RELEASE_EVIDENCE_DIR` / `../release-evidence-temp`; existing draft JSON files are not overwritten unless `RELEASE_EVIDENCE_DRAFT_OVERWRITE=1` is set, these drafts are not consumed by readiness, and they must not be treated as passing evidence. |
| Evidence draft pre-review | `npm run review:release-evidence-drafts` | Checks external draft files before promotion without writing final evidence. | Writes `release-evidence-draft-review-result.json` outside the repository, including `blocking_fields` and `required_operator_inputs`; exit 0 only means drafts are eligible for guarded promotion, not that release readiness is closed. |
| Evidence draft promotion | `npm run promote:release-evidence-drafts` | Fails while drafts still contain draft notices, placeholders, credential-shaped material, or fail the existing design/OTA evidence verifiers. | Reads external drafts, writes `release-evidence-promotion-result.json` or `RELEASE_EVIDENCE_PROMOTION_RESULT_FILE` outside the repository on pass or fail, validates candidate files with the existing design and OTA credential checks, and only then writes `design_handoff_manifest.json` and `ota_credential_rotation_attestation.json` under `RELEASE_EVIDENCE_DIR` / `../release-evidence-temp` outside the repository. |
| OTA attestation creation | `npm run release:create-ota-attestation` | Fails until `OTA_CREDENTIAL_ROTATION_INPUT_FILE` points to a complete controlled external reviewed attestation JSON that passes the same OTA credential verifier. Optional outputs are `OTA_CREDENTIAL_ROTATION_ATTESTATION_OUTPUT` / `OTA_CREDENTIAL_ROTATION_ATTESTATION_FILE` and `OTA_CREDENTIAL_ROTATION_CREATE_RESULT_FILE`. | Writes `release-ota-attestation-create-result.json` or `OTA_CREDENTIAL_ROTATION_CREATE_RESULT_FILE` outside the repository on pass or fail, with `source_mode=external_attestation_input` and input/output SHA-256 fingerprints. Copies only a passing external input file to `OTA_CREDENTIAL_ROTATION_ATTESTATION_OUTPUT`, `OTA_CREDENTIAL_ROTATION_ATTESTATION_FILE`, or `RELEASE_EVIDENCE_DIR/ota_credential_rotation_attestation.json`; input and output must both be outside the repository, must not be draft files, and must not be the same path. The creation result is audit evidence only and does not close readiness without the final attestation and passing isolated gate. |
| Evidence intake behavior contract | `npm run verify:release-evidence-intake` | Temporary external fixtures under the OS temp directory only. | Passes when design and OTA creation scripts accept complete external inputs, reject repository inputs, reject input/output path reuse, reject current connector blocker files as design `source_review.evidence_ref`, reject `sanitized` / `encrypted_archive` as platform credential actions, write create-result files with matching SHA-256 fingerprints, and explicitly do not close readiness. This is a regression guard for the evidence intake path only; it does not write final release evidence and does not replace `npm run review:release-readiness`. |
| Release readiness behavior contract | `npm run verify:release-readiness-contract` | Temporary external fixtures under the OS temp directory only. | Passes only when `review:release-readiness` rejects controlled fixtures where the PR candidate result and external-state result use the same PR number but different head SHA values, or where the local HEAD captured by external-state differs from the final PR head. This is a regression guard for final PR evidence consistency only; it does not write final release evidence and does not replace `npm run review:release-readiness`. |
| Evidence gap pack | `npm run report:release-evidence-gap-pack` | Writes a not-ready gap pack while any blocker remains. | Creates `release-evidence-gap-pack.json` plus the Markdown companion `release-evidence-gap-pack.md` under `RELEASE_EVIDENCE_DIR` / `../release-evidence-temp` outside the repository, selecting the latest generated result files and summarizing missing design handoff, OTA credential rotation, PR candidate state, dirty-worktree category diagnostics, quarantine freshness, draft `blocking_fields`, operator-controlled intake requirements, and staging/isolation bucket entries for `candidate_release_scope`, `needs_explicit_operator_decision`, and `must_remain_local_by_default` without creating fake manifest, attestation, Git staging, or local-state closure evidence. |
| Evidence gap pack contract | `npm run verify:release-evidence-gap-pack` | Passes only when the generated gap pack is structurally usable and non-closing. | Reads `RELEASE_EVIDENCE_GAP_PACK_FILE` / `RELEASE_EVIDENCE_GAP_PACK_MARKDOWN_FILE` or the default controlled evidence paths, rejects repo-internal gap pack files, requires the not-ready source status, design/OTA/PR blocking requirements, readiness close sequence, operator intake inputs, worktree staging/isolation summary, current GitHub connector evidence with preserved open-PR count, and final `npm run review:release-readiness` gate expectation. It does not create or replace final design, OTA, PR, local-state, or release-readiness evidence. |
| Operator intake export | `npm run export:release-operator-intake` | Writes a standalone handoff packet from an existing gap pack. | Reads `RELEASE_OPERATOR_INTAKE_SOURCE_FILE` or `RELEASE_EVIDENCE_DIR` / `../release-evidence-temp/release-evidence-gap-pack.json`, then writes `release-operator-intake-packet.json` and `release-operator-intake-packet.md` outside the repository. This packet only lists real external inputs to collect; it does not replace final design, OTA, PR, local-state, or release-readiness evidence. |
| Operator intake packet contract | `npm run verify:release-operator-intake` | Passes only when the exported operator packet is structurally usable and non-closing. | Reads `RELEASE_OPERATOR_INTAKE_PACKET_FILE` / `RELEASE_OPERATOR_INTAKE_PACKET_MARKDOWN_FILE` or the default controlled evidence paths, rejects repo-internal packet files, requires design, OTA, and final PR/local-state input sections, preserves current GitHub connector and worktree boundary fields, and confirms the packet states it does not close release readiness. |
| Release staged-scope review | `npm run review:release-staged-scope` | Passes when the current Git index is empty or contains only allowed release-scope staged entries. | Writes `release-staged-scope-result.json` to `RELEASE_STAGED_SCOPE_RESULT_FILE` or `RELEASE_EVIDENCE_DIR` / `../release-evidence-temp` outside the repository; fails if staged files include runtime/local artifacts or non-release entries without explicit `RELEASE_STAGED_SCOPE_ALLOW_OPERATOR_DECISION=1`. This is a PR staging guard, not clean-worktree or release-readiness closure. |
| Current evidence refresh | `npm run refresh:release-current-evidence` | Fails while any current release gate fails, but refreshes current JSON/Markdown evidence first. | Runs `verify:release-evidence-intake` and `verify:release-readiness-contract` first, then writes current evidence bundle, draft review, guarded promotion result, PR candidate, staged-scope result, worktree quarantine manifest/review TSVs, readiness, external-state evidence/result, `release-evidence-gap-pack-current.json`, `release-evidence-gap-pack-current.md`, `release-operator-intake-packet-current.json`, and `release-operator-intake-packet-current.md` under `../release-evidence-temp` for audit handoff. The behavior contracts, quarantine bundle, gap pack, and operator intake packet preserve guard/blocker evidence only; they do not close local/PR readiness or final release readiness. |
| Final handoff runner | `npm run review:release-final-handoff` / `npm run release:continue-handoff` | Fails while design handoff or OTA credential rotation evidence is missing. | `review:release-final-handoff` runs the final pre-ready gate sequence in the required order and stops before PR-ready handoff; with `-AfterPrReady`, verifies `review:release-pr-candidates`, reruns staged-scope, collects external-state evidence, and reruns release-readiness against the non-draft final PR head. `release:continue-handoff` can first verify newly supplied controlled evidence before a PR number exists, then reads `selected_release_pr_number` from `release-pr-candidates-result.json` when exactly one viable final PR is selected; `-MarkPrReady` and `-AfterPrReady` still require explicit `RELEASE_PR_NUMBER`. |
| PR ready guard | `npm run release:mark-pr-ready` | Fails while final handoff runner fails, no selected release PR ready target exists, or local `HEAD` does not match the selected PR head. | Runs `review:release-pr-candidates` in explicit draft-allowed ready-transition mode, writes `release-pr-ready-candidates-result.json`, refuses an implicit PR #2 target, requires local `git rev-parse HEAD` to match the selected draft PR head, reruns pre-ready final handoff with external-state explicitly pending, and only marks the selected configured release PR ready for review. This transition is not release-readiness closure; after GitHub Actions remain green, rerun the final handoff with `-AfterPrReady`. | Release readiness | `npm run review:release-readiness` | Fails while any blocker above remains open. | Single release gate for production env, LLM, design handoff, OTA credentials, persistent Codex Security scan artifacts, current staged-scope result, PR candidate result, and current external-state result. By default it reads `RELEASE_EVIDENCE_DIR` or `../release-evidence-temp`, writes `release-readiness-result.json` outside the repository, rejects repo-internal result files, and requires the PR candidate number/head, external-state PR head, and captured local HEAD to match. |
| Status contract | `npm run verify:release-status` | Passes. | Confirms release docs, examples, scripts, and blocker contracts stay consistent. |
| Functional readiness | `npm run review:functional-readiness` | Passes. | Confirms local structural coverage for OTA data, revenue analysis, AI decision, operations management, and investment decision. |
| Issue register | `npm run review:release-issues` | Passes. | Confirms the current release issue register still lists all open blockers and acceptance commands. |
| Non-security review | `npm run review:non-security` | Passes. | Confirms the non-security release review contract still passes. |
| Report/security/finance regression review | `npm run review:report-security-finance` | Passes. | Confirms protected-route tokens are not accepted from URL query parameters, report/admin token leakage regressions stay blocked, and finance simulation math remains stable. |

## Execution Rules

- Do not mark a blocker closed from narrative evidence alone; its isolated command must pass first.
- Do not store real keys, Cookie values, Token values, signatures, Authorization headers, or unredacted sensitive fields in evidence files.
- Do not treat template files as passing production evidence.
- Do not replace the formal Codex Security scan with dependency audits or existing lightweight security scripts; keep the completed scan artifacts passing on the final head.
- Rerun `npm run review:release-readiness` after each blocker is closed.
