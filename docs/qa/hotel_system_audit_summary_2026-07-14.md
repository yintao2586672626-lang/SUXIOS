# 宿析OS 酒店系统符合性审计摘要

审计日期：2026-07-14

## 结论

500 条用例已逐项映射：满足 350、部分满足 74、不满足 31、未验证 19、不适用 26。
排除 25 条完整 PMS 范围外用例和 1 条当前未进入支付卡数据范围的 PCI 条件项后，适用 474 条；已确定项满足率 76.9%，证据覆盖率 96.0%。
宿析OS已具备较强的核心后端与结构守卫，但当前发布门禁和浏览器 E2E 失败，真实携程/美团与生产模型闭环也未形成新鲜证据，因此结论是“功能基础较强，尚未达到完整市场生产就绪”。

## 最高优先级缺口

1. 修复浏览器 E2E 测试账号/种子数据，使登录、酒店、用户、凭据保存与核心页面可真实操作回显。
2. 在授权酒店完成携程与美团同日采集、字段对账、质量状态和收益指标闭环。
3. 补改密撤销会话、登录渐进限流/锁定、高权限 MFA、授权 expires_at 校验。
4. 收紧 CORS 并补 CSP/HSTS/nosniff/frame/referrer 等生产响应头。
5. 落地个人信息告知同意、查阅复制、更正删除、保留期限和跨境/模型传输控制。
6. 补数据库就绪探针、备份恢复/RPO-RTO、负载与灾难恢复实测。
7. 补齐设计交付清单、OTA 凭据轮换证明及新鲜 staged/external-state 发布证据。
