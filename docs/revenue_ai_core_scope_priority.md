# Revenue AI 核心范围优先级

更新日期：2026-06-27

本文件用于把“只保留核心功能”的产品边界固化为当前重构基线。它不是删除旧模块的执行命令；任何删除、隐藏或迁移仍必须按最小改动、旧链接兼容和验证结果逐项执行。

## 主产品边界

主产品名称：宿析OS 收益管理智能体 Revenue AI。

核心日常链路：

```text
携程/美团昨日 OTA 数据
-> 异常判断
-> 竞对对比
-> 动态定价建议
-> 今日执行动作
-> 明日效果复盘
```

第一版只解决早晨高频问题：昨天 OTA 表现如何、数据是否可信、哪些数据缺失、是否具备调价判断条件、今天谁去执行、明天如何用真实证据复盘。

## P0 保留

| 模块 | 当前入口 | 保留原因 | 边界 |
|---|---|---|---|
| Revenue AI 总览 | `收益管理智能体总览` / `compass` | 每日收益管理主入口 | 读取 `/api/revenue-ai/overview`，不伪造成功，不直接写 OTA |
| OTA 数据健康 | `线上数据 -> 数据健康总览` | 判断携程/美团数据是否可信 | 登录失效、缺字段、接口未命中必须展示真实原因 |
| 携程 eBooking | `线上数据 -> 携程ebooking` | 核心 OTA 数据源 | 优先经营概况、流量、订单、竞对价格 |
| 美团 eBooking | `线上数据 -> 美团ebooking` | 核心 OTA 数据源 | 优先流量、订单、价格库存；广告为可选 |
| 运营执行 | `运营执行` | 承接调价建议、执行证据和 ROI 复盘 | 人工审核、人工执行、证据复盘；不自动改价 |
| 系统设置 | `系统设置` | 门店、权限、模型、知识库等基础配置 | 普通经理不可见超级管理员工具箱 |

## P1 支撑

| 模块 | 当前状态 | 后续动作 |
|---|---|---|
| `app/service/RevenueAiOverviewService.php` | Revenue AI 首页聚合服务 | 继续承载只读聚合、缺口原因、执行复盘摘要 |
| `public/revenue-ai-static.js` | Revenue AI 首页展示 helper | 继续拆分展示整形逻辑，保持 `public/index.html` 变薄 |
| `app/controller/OnlineData.php` | 最大 OTA 采集控制器候选 | 后续按携程/美团、手动/自动、数据模块逐步拆 service |
| `public/index.html` | 最大前端入口候选 | 后续按页面/helper 边界拆，不重启 Vite，不改全局导航语义 |

## P2 下沉

这些模块保留旧 path 和旧能力，但不再挤占主工作台；统一挂在“全生命周期辅助”下，默认不作为 Revenue AI 首页闭环的一部分：

- 投资、筹建、战略推演、量化模拟、可行性报告
- 开业准备、开业检查
- 扩张、市场评估、标杆选模、协同提效
- 转让、资产定价、时机推演、转让数据看板
- 图片优化助手
- 酒店 AI 工具箱

## 不做

- 不重写整个系统。
- 不删除旧模块、旧 path 或历史数据。
- 不新增生产依赖。
- 不改数据库结构。
- 不把 OTA 渠道口径包装成全酒店经营口径。
- 不自动写携程/美团价格。
- 不用空数据、默认值或宽泛 catch 掩盖采集失败、登录失效、缺字段或接口未命中。

## 验收线

自动验收由 `npm.cmd run verify:core-scope` 覆盖，至少检查：

- 一级导航只有 `收益管理 / 线上数据 / 运营执行 / 系统设置`。
- 默认首页内部 path 仍是 `compass`，用户可见名称是“收益管理智能体总览”。
- “全生命周期辅助”位于收益管理二级入口内，旧二期模块只下沉不删除。
- 普通经理不可见 `agent-center` 超级管理员工具箱。
- Revenue AI 首页使用 `/api/revenue-ai/overview` 和 `public/revenue-ai-static.js`。
- `ota_room_revenue`、`ota_room_nights`、`ota_adr`、`ota_contribution_revpar` 继续显式区分 OTA 渠道口径与全酒店口径；缺全酒店可售房晚时保留 `available_room_nights_missing`。
- 调价建议只能走人工审核、转运营执行意图、执行证据和 ROI 复盘证据，不直接写 OTA。
