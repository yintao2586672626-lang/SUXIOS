## 知识分层

### 1. 通用方法 `generic_methodology`

可以跨酒店复用，但必须绑定当前门店的真实数据：

- 收入变化的销量与价格桥接。
- 渠道结构从“占比”升级到“量、价、成本、取消、提前期和房型”的联合判断。
- 房型角色识别与价格梯度设计。
- 淡旺季、目标区间和渠道占比的可行性校验。
- 7/14/30 天 OTB、Pickup 和剩余库存触发框架。
- 事实、推导、假设、动作、保护条件和复盘指标的建议卡契约。

### 2. 经验规则 `experimental_rule`

只作为待校准模板：

- OTB 达到某个阈值后关低价、提价或收紧库存。
- 某渠道承担价格锚点、临期补量或高价值承接。
- 某房型承担引流、升级、价格保护或场景溢价。

经验阈值必须标明门店、周期、样本和复盘结果；缺少历史 Pickup 曲线时不得写成预测模型结论。

### 3. 案例参考 `case_reference`

仅在调用方显式提供 `case_key=moke_yuexiang_2026_h2` 时返回，用于教学、计算复核和规则设计，不进入默认知识上下文。

### 4. 禁用边界 `boundary`

- 不把 OTA 渠道数据扩大为全酒店经营事实。
- 不把渠道结构变化直接写成 ADR 下降的确定原因。
- 不把特殊时期极端值直接删除或并入常态基线。
- 不把经验阈值包装成已验证模型。
- 不让知识内容直接触发 OTA 改价、关房、开房或库存写回。

## 通用诊断方法

### 收入变化桥接

至少拆成：

```text
销量影响 =（本期间夜 - 对比期间夜）× 对比期 ADR
价格影响 = 本期间夜 ×（本期 ADR - 对比期 ADR）
收入变化约等于销量影响 + 价格影响
```

使用要求：

- 两期收入、间夜和 ADR 必须是同一范围、同一日期基础和同一收入定义。
- 结果是结构桥接，不是因果识别。
- 展示值经过四舍五入时应保留尾差说明。
- 若要判断渠道或房型导致的价格变化，还需继续拆渠道 ADR、房型结构和佣金后净收入。

### 渠道收益判断

渠道占比只回答“量从哪里来”，不能独立回答渠道效率。至少同时比较：

- 绝对间夜与间夜占比。
- 房费收入与收入占比。
- 毛 ADR 与佣金后净 ADR。
- 取消率、净 Pickup、提前期和平均入住天数。
- 房型结构、促销价格计划和高需求日低价占用。

推荐判断：

> 渠道结构变化与收益变化同时出现时，可以列为归因假设；没有净 ADR、佣金、取消和房型证据时，不得写成确定原因。

### 房型角色

| 角色 | 识别信号 | 经营任务 | 保护条件 |
| --- | --- | --- | --- |
| 引流房 | 需求广、价格敏感、承担入口流量 | 低峰限量补量 | 旺日不得无边界外溢低价 |
| 收益发动机 | 间夜与收入贡献高、需求稳定 | 保持转化并逐步修复价格 | 观察提价后的转化和取消 |
| 升级承接房 | 与基础房有清晰支付理由 | 设计价差与升级链路 | 不能只改名称而无产品差异 |
| 价格保护房 | 高需求日仍能维持较高 ADR | 守价、保护库存 | 不用低价房销量逻辑考核 |
| 场景溢价房 | 影音、家庭、茶室、麻将等场景明确 | 用权益、时段和场景组合溢价 | 权益成本和机会成本要可复核 |

房型、价格计划、促销套餐和渠道库存必须分开建模；“关闭最低房型”不能代替准确的执行对象。

### 淡旺季与目标校验

- RevPAR 可以作为周期判断主指标，ADR 与出租率作为辅助指标。
- “历史均值上下浮动固定比例”只能作为启发式规则。
- 更可靠的基线还应考虑月份同期、星期结构、节假日、事件、竞对和历史 Pickup。
- 月度目标应区分保底、预算、冲刺三个场景，并能回算到房量、ADR、出租率和收入。
- 多渠道占比区间必须联合检查：最终分配合计应为 100%，不能分别机械兑现每个上限或下限。

### 特殊时期与异常值

- 节假日、演唱会、考试、会展和异常高价订单应保留事件标签。
- 可以从常态基线中剥离，但不能删除原始事实。
- 必须核对样本数、收入、入住日期、房型、渠道和退款取消。
- 小样本极端 ADR 只能作为待复核提示。

### 7/14/30 天 OTB 与 Pickup

建议规则结构：

```text
观察窗口 + 当前 OTB + 历史 Pickup 分位 + 剩余库存
+ 当前价格与竞对 + 取消修正
→ 动作候选 + 保护条件 + 复盘时间
```

经验阈值使用规则：

- 没有历史 Pickup 曲线时标记 `experimental_rule`。
- 达到阈值不等于自动执行，只产生待人工复核动作。
- 先调整价格计划和低价库存，再判断是否调整房型基础价。
- 每次动作记录执行前后 ADR、净 RevPAR、转化、取消、Pickup 和剩余库存。
- 效果不达标或出现保护条件时停止、回滚或重新校准。

## 建议卡调用契约

后续功能使用本知识时，一条建议至少包含：

1. `scope`：OTA 渠道、全酒店、房型、渠道或价格计划。
2. `business_date/date_basis`：经营日期和日期口径。
3. `facts`：已验证事实及来源。
4. `derived_metrics`：公式、输入和结果。
5. `hypotheses`：尚未证实的解释。
6. `missing_evidence`：缺失数据和补数入口。
7. `target_object`：渠道、房型、价格计划、套餐或库存。
8. `trigger`：观察窗口、阈值、Pickup、剩余库存。
9. `action`：人工可执行动作。
10. `guardrails`：底价、库存、范围和禁止条件。
11. `owner/review_at`：责任人与复盘时间。
12. `expected_metric`：净 RevPAR、ADR、转化、取消或 Pickup。

没有事实和前置条件时，只返回补数要求，不生成假建议。

## 墨客悦享案例参考

案例键：`moke_yuexiang_2026_h2`。

### 来源事实

- 2025 年上半年收入 1,215,301 元；2026 年上半年收入 1,099,607 元，下降 9.5%。
- 出租间夜从 8,019 降至 7,839，下降 2.2%。
- ADR 从 151.55 元降至 140.27 元，下降 7.4%。
- 展示出租率从 96.7% 降至 94.3%；按展示值为下降 2.4 个百分点，原材料写 2.3，需用底层精度复核。
- 携程间夜从 4,752 降至 4,139；美团间夜从 2,620 升至 3,113。
- 2026 年 2 月线下 ADR 700.68 元被材料解释为春节特殊时期，仍需订单数、收入和日期证据验证。
- 2026 年下半年收入目标约 123.42 万至 137.47 万元；属于案例规划，不是其他门店目标。

### 派生复算

- 收入减少 115,694 元。
- 简化桥接的销量影响约 -27,279 元，价格影响约 -88,424 元。
- 价格项约占收入下降的 76.4%，销量项约占 23.6%；这是结构解释，不是因果结论。
- 携程间夜占比约从 59.26% 降至 52.80%，下降 6.46 个百分点。
- 美团间夜占比约从 32.67% 升至 39.71%，上升 7.04 个百分点。
- 渠道变化与 ADR 下降同时发生，但缺少渠道净 ADR、佣金、取消率、提前期和房型结构，不能定责。

### 案例允许用途

- 演示收入桥接计算。
- 演示渠道结构只能作为归因假设。
- 演示如何把房型角色、渠道角色和 OTB 阈值组织成经营计划。
- 验证建议卡是否区分事实、推导、假设和补数要求。

### 案例禁止用途

- 作为其他酒店当前经营事实。
- 复制渠道占比、价格步长或月度收入目标到其他酒店。
- 用案例阈值直接生成自动调价或库存写回。
- 将墨客悦享的线下、OTA和全酒店数据范围混为一体。

## 调用方式

知识通过 `RevenueOperationsKnowledgeService` 只读调用：

```php
$context = (new RevenueOperationsKnowledgeService())->load([
    'hotel_id' => $hotelId,
    'types' => ['使用边界', '收入变化诊断', '建议卡契约'],
]);
```

默认不会返回案例事实。只有显式指定案例键时才返回对应 `case_reference`：

```php
$caseContext = (new RevenueOperationsKnowledgeService())->load([
    'hotel_id' => $hotelId,
    'case_key' => 'moke_yuexiang_2026_h2',
]);
```

调用方必须检查 `status` 与 `data_gaps`；`missing_table`、`empty` 或 `partial` 不得被包装成知识已就绪。
