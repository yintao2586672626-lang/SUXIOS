## 核心框架

OTA 运营按四条主线组织：

| 主线 | 目标 | 关键内容 |
| --- | --- | --- |
| 平台运营 | 明确平台、渠道和合作策略 | 携程、美团、飞猪、分销、短租、境外平台；直营、代理、独家、精选、全渠道 |
| 产品运营 | 提升产品可售能力和转化 | 单酒、钟点房、会议房、套餐、景酒、机酒、门票、餐饮、SPA、租车；房型、价格、退改、卖点 |
| 活动运营 | 获取活动、标签和广告流量 | 免费促销、场景促销、付费广告、平台活动、店铺活动、节假日活动 |
| 数据运营 | 用数据诊断并闭环执行 | 订单、间夜、营业额、取消、佣金、流量来源、转化率、平台指数、竞对、用户画像 |

## 平台与渠道定位

OTA 的基本商业模式是线上曝光展示，订单产生后按佣金或广告方式收费。平台差异决定运营策略：

| 平台/渠道 | 典型特征 | 适配方向 |
| --- | --- | --- |
| 携程系 | 中高端、商旅、体量较大，挂牌和服务质量分影响流量 | 深耕挂牌、保留房、服务分、金字塔广告位、分销维护 |
| 美团点评系 | 本地流量、年轻用户、价格敏感，HOS 和推广通影响流量 | 维护 HOS、预留房、金币/冠级、点评、推广通 |
| 飞猪 | 年轻用户、信用住、活动和 MIC 指数 | 完善基础信息、图片、活动、可退改、闪电确认、MIC |
| 分销/短租/境外渠道 | 补充曝光或特定客群 | 控制价格一致性、库存成本和管理复杂度 |

渠道策略分三类：

- 全渠道运营：最大化曝光，但价格和库存管理成本高。
- 精选渠道运营：适合多数门店，选主渠道深耕，辅渠道补充。
- 独家渠道运营：换取平台支持，但会损失其他平台曝光，并与平台强绑定。

## 流量体系

OTA 流量主要来自搜索、产品、活动广告、内容、自主访问和复购。

| 流量类型 | 关键入口 | 可操作点 |
| --- | --- | --- |
| 搜索流量 | 城市空搜、筛选关键词、个性化智能排序、标签搜索 | 提升排名、完善标签、维护价格库存、提升点评分、参与活动 |
| 产品流量 | 全日房、钟点房、会议房、民宿、套餐、景酒、机酒、门票打包 | 扩展可售产品，覆盖不同出行场景 |
| 活动促销流量 | 特价、限时、提前订、连住、会员、新客、节假日大促 | 计算活动成本，避免提价打折和线下补差价 |
| 付费流量 | 携程金字塔、直通车、点金手、美团推广通、钻石展位 | 设置预算、出价、时段、人群和场景，记录转化效果 |
| 内容流量 | 攻略、问答、点评、短笔记、短视频、旅拍、旅行笔记 | 用真实图文和问答承接酒店卖点，可嵌入酒店链接 |
| 自主访问流量 | 微信公众号、头条、百家号、小红书、知乎、微博、问答社区 | 一文多发，内容带预订链接，不放违规联系方式 |

## 转化漏斗

| 阶段 | 用户路径 | 关键影响因素 | 优先动作 |
| --- | --- | --- | --- |
| 点击转化 | 搜索页进入详情页 | 搜索排名、首图、名称、点评、位置、最低价、标签、近期成交、竞对 | 调整首图和起价房型，提升排名、点评分和正向标签 |
| 预订转化 | 详情页进入订单页 | 图片、设施、简介、房型、点评、问答、竞对、价格梯度、退改政策 | 完善详情页内容、房型名称、床型、价格、退改、发票和支付说明 |
| 支付转化 | 填写页进入提交/支付 | 支付方式、取消政策、发票类型、用户不确定因素 | 明确到店付/在线付/信用住、免费取消/限时取消/不可取消 |

诊断逻辑：

- 点击转化低：看排名、首图、起价、点评分、标签、竞对价格。
- 预订转化低：看房型信息、图片完整度、退改政策、问答、点评回复。
- 支付转化低：看支付方式、发票、取消政策、订单填写风险点。

## 内容资产

| 资产 | 标准动作 | 运营价值 |
| --- | --- | --- |
| 图片 | 首图接近 3:4；类别完整；实景优先，可测试创意图；周期性更换并记录转化 | 影响点击转化和用户第一印象 |
| 房型信息 | 名称控制在合理长度；包含客群、功能、卖点、床型；价格区间和梯度清晰 | 承接筛选、价格感知和预订转化 |
| 酒店简介 | 分段式、套餐化、场景化；写设计、早餐、设施、服务、周边、交通、Q&A | 打消疑虑，强化卖点 |
| 特色卖点 | 在简介、图文秀、房型、问答、点评回复、礼盒、店内商城等位置展示 | 提升差异化和内容转化 |
| 标签 | 由基础信息、点评关键词、活动、业务、房型、设施和人工规则生成 | 提升搜索、筛选、点击和预订转化 |

简介可按“五觉场景法”组织：视觉、听觉、嗅觉、触觉、味觉。内容必须对应真实设施和服务，不写无法交付的卖点。

## 点评与口碑

好评价值：

- 提升流量、排序、标签曝光、转化率和价格承受力。
- 好评是价格基础，也是竞对比较中的信任资产。

好评获取方式：

- 高于预期的入住体验。
- 入住后提醒点评。
- 活动或礼物激励，但不得诱导虚假评价。
- 回复中植入真实卖点、活动预告和特色内容，避免机械套模板。

差评常见因素：

- 卫生、隔音、热水、网络、设施设备、服务、性价比、纠纷。
- 特价房、高价房、临街房、不可取消房型更容易形成预期落差。
- 到店无房、确认后满房、确认后涨价会触发系统差评或平台处罚。

差评处理原则：

- 先判断平台规则：可删除、可修改、可申诉、只可回复。
- 申诉必须保留截图、订单、沟通记录等证据。
- 回复结构：诚挚道歉、解释说明、后续补救、明确改进。
- 不使用第三方违规删评，不伪造好评，不把历史删评路径写成当前可执行规则。

## 房态库存与日常维护

保留房/预留房是平台可自由售卖的一定数量库存，常用于提升平台分、排序、奖励和确认效率。维护规则：

- 每个主要房型尽量有保留房/预留房。
- 热销房型和高产房型设置更多库存。
- 现付和预付都要覆盖。
- 保留时间内不要随意关闭未售完库存。
- 房型满时优先升级、调配其他渠道客人或安排其他店，避免拒单和到店无房。
- 酒店满房时必须提前处理房态，不用假库存掩盖问题。

日常维护清单：

| 模块 | 检查点 |
| --- | --- |
| 订单 | 新订单、修改单、取消单、延住单；携程/美团 5 分钟确认，飞猪 1 分钟确认 |
| 价格 | 外网价格体系、远期价格、当天价格、节假日价格、竞对价格、低价房型 |
| 房态 | 普库库存、保留房库存、剩余库存、节假日库存、线下渠道占用 |
| 点评 | 好评回复、差评处理、关键词分析、正向和负向评价点 |
| 问答 | 日常问答回复、IM 及时回复、自动回复内容 |
| 活动 | 促销、付费广告、参与、修改、取消、有效时段和无效流量剔除 |
| 图片 | 酒店首图、房型首图、类别补齐、周期测试 |
| 数据 | 流量、转化率、用户、营业额、订单量、间夜、违规指标 |

## 数据诊断闭环

数据运营要回答四个问题：

1. 哪个平台、渠道或入口贡献了订单、间夜和营业额。
2. 流量低是搜索、筛选、直搜、内容、活动还是付费入口的问题。
3. 转化低发生在点击、预订还是支付阶段。
4. 当前平台分和违规项卡在哪些维度。

核心指标清单：

- 销售数据：营业额、间夜量、订单量、取消量、平台佣金、平台贡献度。
- 流量数据：曝光、访客、城市搜索、筛选关键词、直搜、内容流量、流量来源结构。
- 转化数据：点击转化率、预订转化率、支付转化率。
- 运营指标：携程服务质量分、美团 HOS、飞猪 MIC、挂牌/挂冠、广告消耗。
- 违规指标：到店无房、确认后满房、确认后涨价、拒单、逃单、价格倒挂、刷单。
- 用户数据：性别、年龄、偏好、浏览时间、预订时间、入住天数、提前预订天数。
- 竞对数据：价格、点评、排名、活动、标签、库存、图片和卖点。

闭环动作：

| 诊断结果 | 动作 |
| --- | --- |
| 高流量高转化 | 维持并扩大有效库存、活动和广告预算 |
| 高流量低转化 | 优化首图、详情页、房型、价格、退改、点评和问答 |
| 低流量高转化 | 提升排名、标签、活动、付费投放和内容分发 |
| 低流量低转化 | 先修基础信息、价格库存、点评分和平台分，再扩流量 |
| 平台分低 | 拆到确认率、保留房、无缺陷订单、信息完整度、点评、违规项逐项修复 |
| 竞对强 | 对比价格、设施、图片、标签、活动和点评，找可执行差异点 |

## 宿析OS落地规则

- 作为知识库：沉淀到 `knowledge_units`、`knowledge_chunks`、`knowledge_base`，供 Agent 检索。
- 作为指标口径：映射到已有 `online_daily_data.raw_data`、日报、收益分析、竞对分析，不因本资料直接新增表。
- 作为运营任务：转成可执行任务清单时必须绑定平台、酒店、日期、指标、当前值、目标值、动作和验证方式。
- 作为 AI 决策依据：只输出基于现有数据可验证的建议；缺少字段时返回“待补数据/不可计算”，不写兜底假值。
- 作为平台规则：所有具体条件、分值、广告位、删除/申诉路径和联系方式执行前必须复核当前平台后台。
