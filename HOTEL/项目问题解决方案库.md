### 问题 #001：PHP 服务器启动失败 - 找不到 router.php

| 项目 | 内容 |
|------|------|
| **问题描述** | 运行 `run.bat` 启动 PHP 内置服务器时报错：`Failed opening required 'router.php'` |
| **错误信息** | `Fatal error: Failed opening required 'router.php' (include_path='C:\xampp\php\PEAR')` |
| **问题类型** | 路径问题 |
| **根本原因** | `run.bat` 中使用 `%~dp0public\router.php` 相对路径，PHP 解析路径错误 |
| **解决方案** | 使用绝对路径启动，先切换到 public 目录再启动 |
| **正确命令** | ```cmd<br>cd /d c:\Users\Admin\Desktop\JDXM\JDSJ\HOTEL\public<br>C:\xampp\php\php.exe -S 0.0.0.0:8081 router.php<br>``` |
| **相关文件** | `run.bat`, `start.bat` |
| **预防措施** | 使用 `start.bat` 脚本启动，避免手动输入命令导致路径错误 |

### 问题 #002：端口 8080 被占用

| 项目 | 内容 |
|------|------|
| **问题描述** | 启动 PHP 服务器时端口 8080 已被多个进程占用 |
| **错误信息** | 无法绑定端口，或启动后无法访问 |
| **问题类型** | 端口冲突 |
| **根本原因** | 之前多次启动的 PHP 进程未正常关闭，导致端口冲突 |
| **解决方案** | 1. 换用其他端口（如 8081、8090）<br>2. 清理残留进程 |
| **验证命令** | `netstat -ano \| findstr :8080` |
| **清理命令** | `taskkill /F /IM php.exe` |
| **预防措施** | 1. 使用固定端口范围（8081-8090）<br>2. 启动前检查端口占用<br>3. 及时清理残留进程 |

### 问题 #003：ngrok 获取不到公网 URL

| 项目 | 内容 |
|------|------|
| **问题描述** | ngrok 启动后无法获取到公网访问地址 |
| **错误信息** | `无法连接到远程服务器` 或 Web 界面 4040 端口无法访问 |
| **问题类型** | 服务启动问题 |
| **根本原因** | ngrok 启动后需要等待几秒才能生成隧道，Web 界面需完全启动 |
| **解决方案** | 1. 等待 3-5 秒后通过 API 获取<br>2. 直接查看 ngrok 窗口的 `Forwarding` 地址 |
| **获取命令** | ```powershell<br>powershell -Command "Start-Sleep -s 4; $r = Invoke-WebRequest 'http://127.0.0.1:4040/api/tunnels' -UseBasicParsing; ($r.Content \| ConvertFrom-Json).tunnels[0].public_url"<br>``` |
| **预防措施** | ngrok 免费版每次重启 URL 会变，需要重新获取并记录 |
