# 📋 问题追踪系统 (Issue Tracker)

自动追踪项目问题，当同一问题发生 **4 次** 以上时自动归档到解决方案库。

## 🎯 核心功能

- ✅ **自动计数**: 记录每个问题的发生次数
- ✅ **阈值检测**: 达到 4 次自动触发归档
- ✅ **自动归档**: 自动添加到解决方案库
- ✅ **统计报表**: 查看问题分布和趋势

## 📁 文件结构

```
.issue_tracker/
├── issues.json          # 问题数据库
├── track_issue.php      # 核心追踪脚本
├── README.md           # 本文档
└── archive/            # 归档的问题详情
```

## 🚀 使用方法

### 方式一：使用批处理（推荐）

```cmd
:: 记录问题
track_issue.bat "数据库连接超时" "数据库"
track_issue.bat "页面加载慢" "性能" "Timeout after 30s"

:: 查看统计
track_issue.bat stats

:: 列出所有问题
track_issue.bat list
```

### 方式二：直接使用 PHP

```cmd
:: 记录问题
php .issue_tracker\track_issue.php track "问题标题" "类别" "错误信息"

:: 查看统计
php .issue_tracker\track_issue.php stats

:: 列出所有问题
php .issue_tracker\track_issue.php list
```

## 📊 问题类别

| 类别 | 说明 |
|------|------|
| 运行环境 | PHP、服务器、端口等问题 |
| 数据库 | MySQL、SQLite、连接等问题 |
| 代码 | PHP语法、逻辑错误等 |
| 网络 | API、跨域、ngrok等问题 |
| 数据 | 数据解析、导入导出等 |
| 权限 | 认证、授权等问题 |
| 性能 | 加载慢、内存溢出等 |

## ⚙️ 自动归档流程

```
问题发生 → 记录到 issues.json → 计数+1
                                ↓
                          计数 >= 4?
                                ↓
                    是 → 自动添加到解决方案库
                                ↓
                    生成待完善的问题条目
```

## 📝 归档后操作

当问题自动归档到解决方案库后，需要**手动补充**以下信息：

1. **根本原因分析**
2. **详细解决步骤**
3. **预防措施**
4. **相关文件**

## 🔄 与解决方案库集成

自动归档的问题会插入到 `项目问题解决方案库.md` 的合适位置，带有标记：

```markdown
### 问题 #020：xxx问题 【自动归档】

| 项目 | 内容 |
|------|------|
| **根本原因** | ⚠️ 待补充 - 请分析并填写 |
| **解决方案** | ⚠️ 待补充 - 请填写详细解决步骤 |
| **预防措施** | ⚠️ 待补充 - 请填写如何避免再次发生 |
```

## 📈 统计数据

运行 `track_issue.bat stats` 可查看：
- 总问题数
- 已解决问题数
- 自动归档问题数
- 待处理阈值问题数

## 💡 最佳实践

1. **及时记录**: 遇到问题立即记录，不要等多次发生后再补
2. **准确分类**: 选择正确的问题类别，便于后续统计
3. **详细描述**: 尽量提供错误信息和上下文
4. **定期整理**: 定期查看 `pendingIssues`，完善自动归档的问题

## ⚠️ 注意事项

- 阈值默认为 **4 次**，可在 `track_issue.php` 中修改 `$threshold`
- 自动归档只执行一次，同一问题不会重复归档
- 归档后的问题仍会继续计数，用于统计
