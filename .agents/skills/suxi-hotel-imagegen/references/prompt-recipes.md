## 共用骨架

只使用与当前任务有关的行：

```text
Use case: hotel-photo-edit
Mode: <ota_fact_edit | brand_creative | video_storyboard>
Asset: <hotel / room / space / target channel>
Input images: <Image 1: edit_target; Image 2: style_reference>
Primary request: <本轮唯一修改>
Scene: <客房/卫生间/公区/外立面>
Lighting: <自然光、室内灯光或创意氛围>
Change only: <允许变化>
Must preserve: <酒店事实不变量>
Avoid: <伪影、事实漂移、文字错误>
Output label: <模式标签>
```

所有编辑提示都重复：`Change only X; keep all listed hotel facts unchanged.`

## 客房

### 自然光 + 被子平整

```text
平衡现有自然光和白平衡，压回窗户高光但不生成原图不存在的窗景。整理当前被子的大褶皱，使床面平整、干净、仍保留真实布料纹理、缝线、图案和自然压痕。只改变光线与被子表面；保留房间几何、床型、床数、床架、枕头数量、家具、门窗、设施、固定材质和镜头位置。避免塑料布料、重复纹理、床沿融化、空间扩容和过度 HDR。
```

### 自然光 + 被子蓬松

```text
优化现有自然采光，使暗部有层次且窗外不过曝。轻度整理现有被子，让它自然蓬松而不是新增体积；保留被芯边界、布草纹理、床型、枕头数量和真实铺床方式。不得改变家具、设施、房间面积、窗景或结构。
```

### 室内打光 + 被子平整/蓬松

在对应修床提示前加：

```text
只优化现有灯具产生的室内光，统一混合色温，恢复暗部层次，降低窗外高光；不要新增灯具、改变光源方向或制造原现场不可能的阳光。
```

## 卫生间与公区

### 卫生间事实精修

```text
修正色温、局部过曝、噪点和轻微镜面眩光；仅移除已确认的临时垃圾或拍摄器材。保留洁具、龙头、玻璃隔断、镜面、灯具、地漏、扶手、用品和标识的数量与位置。镜中反射、玻璃边缘、光源和阴影必须物理一致。避免删除固定设施、凭空补墙、重复瓷砖和镜像文字。
```

### 公区事实精修

```text
平衡大堂/餐厅/走廊的曝光与色温，恢复窗户和灯光层次，只清理已确认的临时杂物。保留家具布置、通道宽度、消防设施、导视、品牌标识和永久装饰。不得用客房修图逻辑重排公区家具。
```

## 外立面

### 白天/夜晚基础精修

```text
校正轻微透视、曝光、白平衡、噪点和楼体表面的小型临时污迹。只移除已确认的临时车辆、垃圾或拍摄器材；保留楼层、窗户、入口、招牌、立面材质、道路、永久高架、电线杆、绿化和周边建筑。文字和 Logo 必须原样保留。
```

### 真实亮灯

只有酒店确认相应区域实际可亮灯时使用：

```text
增强原图中真实已开启灯光的可见度与暖冷层次，不点亮原图无法确认或现场不存在的窗户/招牌灯。保留窗格、楼层、招牌文字和外立面结构。
```

## 清晰度与通用编辑

### 放大

```text
提高输出像素和可见细节，保留真实纹理、文字、边缘和几何；不要重设计材质、发明文字或增加设施。输出后核验实际像素。标签写“算法放大到 <尺寸>”，不得写“原生 8K”。
```

### 局部编辑

```text
只修改明确选区内的 <对象/问题>，保持选区外像素语义、构图、光线和酒店事实不变；匹配周边纹理、透视、阴影和反射，不产生修补边界。
```

### 多图参考

```text
Image 1 是唯一 edit_target；Image 2 仅为 <style_reference/lighting_reference/composition_reference>。只迁移指定的色调、光线或构图节奏；禁止迁移 Image 2 的家具、窗景、设施、装修、Logo、文字、空间结构或房型特征。
```

## 创意氛围

以下都使用 `brand_creative` 和 `brand_creative_not_for_ota`：

### 日落/蓝调/日转夜/阳光

```text
把现有酒店图片制作成明确标注的品牌创意效果图：调整为 <日落/蓝调/夜景/阳光> 氛围，保持建筑或房间主体可识别，避免新增未请求设施和错误文字。允许光线、天空和时间氛围变化；不得描述为现场实景或 OTA 房型事实。
```

### 民宿风

```text
基于现有带床房间制作民宿风概念图，借鉴自然材质、柔和布艺和温暖光线；所有新增软装或材质变化都属于创意示意，不代表真实在售房型。输出必须带 brand_creative_not_for_ota。
```

### 扩图

```text
向 <方向> 扩展画布并延续透视、光线和材质。新生成区域属于创意推断，不得用来证明真实面积、设施、景观或空间关系。保留原始有效画面不变，并标明扩展区域非实拍。
```

### 地拍转高视角

```text
制作高视角品牌概念图，保持建筑主体身份和立面特征；新视角、被遮挡区域与环境补绘均为生成推断。禁止用于 OTA 实景、位置或周边环境证明。
```

## 视频分镜

无已授权视频提供方时只交付：

1. 每张输入图的角色、酒店与空间绑定。
2. 画面顺序、时间码、推拉摇移动效和转场。
3. 竖屏/横屏构图建议；规格未核实时写 `channel_spec_unverified`。
4. 字幕、Logo 和安全区建议。
5. 每个镜头的图生视频提示词与必须保留事实。
6. 封面提示词、音乐授权状态和 `storyboard_only` 标签。

视频中的每张酒店图标为 `scene_source`，不是多个静态编辑 `edit_target`；全部 `scene_source` 必须属于同一家酒店，每个镜头只从自己的图片读取事实。酒店绑定未知时只交内部草案，确认混入其他酒店时拆分任务或阻塞合并。

不要把静态分镜方案声称为已生成视频；不要生成虚构房间视角或跨空间连续镜头来误导空间关系。
