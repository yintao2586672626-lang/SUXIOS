# 酒店图片视觉回检

## 目录

- 回检顺序
- 场景检查
- 失败处理

## 回检顺序

1. 以原始 `edit_target` 为事实基准，并排查看原图和输出图。
2. 确认 `change_only` 已完成且没有额外改动。
3. 逐项核对 `must_preserve`，不要只判断“好不好看”。
4. 在 100% 视图检查局部，再在目标渠道缩略预览检查整体。
5. 读取实际文件尺寸；“8K”必须由像素证明。
6. 不能重新打开输出时，结果只能是 `not_visually_verified`。

## 通用检查

- 酒店、门店、房型/空间绑定未变化。
- 墙体、门窗、床、家具、设备、洁具和设施没有增删或变形。
- 窗景、楼层、朝向、永久遮挡和周边环境没有被伪造。
- Logo、门牌、招牌、电视文字、价格、数字和消防标识没有乱码。
- 烟感、插座、开关、空调、投影和固定灯具仍在正确位置。
- 镜面、玻璃、反射、阴影和光源互相一致。
- 无重复纹理、融化边缘、幽灵物体、塑料布料、过度锐化或 HDR 光晕。
- 原图文件未被覆盖；输出是新版本。

## 场景检查

### 客房

- 床型、床数、枕头数量和床架没有变化。
- 被子整理没有改变布草图案、材质或真实尺寸。
- 广角和透视没有夸大面积。
- 窗外高光恢复没有生成新景观。

### 卫生间

- 洁具、玻璃、镜子、五金和用品数量正确。
- 镜中反射与主体对应。
- 瓷砖纹理不重复或断裂。

### 公区

- 家具、通道、导视、消防和固定装饰完整。
- 临时杂物与真实服务用品已区分。

### 外立面

- 楼层、窗格、入口、招牌和立面结构正确。
- 永久道路、高架、电线杆、绿化和邻楼未被当作杂物删除。
- 亮灯不产生整齐复制、无来源光或错误招牌文字。

## 失败处理

| 发现 | 状态 | 下一步 |
| --- | --- | --- |
| 请求变化未完成 | `review_required` | 只修该变化 |
| 事实发生漂移 | `review_required` | 回到原图，恢复受保护事实 |
| 文字/Logo 失真 | `review_required` | 局部修复或人工蒙版恢复 |
| 输出不可打开 | `not_visually_verified` | 不声称通过，保留待检 |
| 工具失败 | `failed` | 报真实失败阶段，不写假成功 |

AI 回检不能替代现场核验。OTA 候选稿必须由熟悉酒店现场的人确认酒店、房型、设施和景观一致性。
