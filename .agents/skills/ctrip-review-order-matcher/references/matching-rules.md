# 点评订单匹配规则

## 数据源

- 点评：`review_id/commentId`、`publish_time`、正文、点评房型、入住月份/入住离店日期、可见订单号或渠道订单号。
- 携程订单池：渠道订单号、可选PMS内部订单号、渠道、房型、入住离店、状态、金额、详情复核状态。
- PMS不是硬依赖。没有PMS字段时照常评分，但金额、内部订单号或详情证据缺失会阻止高置信。
- 携程脱敏昵称不参与订单唯一确认，也不用于身份反查。

## 分值

| 维度 | 分值 | 规则 |
|---|---:|---|
| 强标识 | 100 | 点评订单号或渠道订单号与订单渠道标识完全一致 |
| 房型 | 0-30 | 明确映射或标准化同名30；主体名25；关键词20-25；无映射0 |
| 日期 | -30至30 | 早于离店日14:00为-30；离店后0-3天30；4-7天22；8-14天12；更远4 |
| 状态 | 0-15 | 已退房/已离店/已完成15；进行中/已预订6；取消/关闭/NoShow 0 |
| 金额 | 0-10 | 大于0为10；缺失4；0元0 |
| 内容 | -10至10 | 内容与房型吻合4或10；硬冲突-10 |
| 唯一性 | 0-5 | 同窗口同映射房型只有一个清晰候选5 |
| 重复命中 | -15 | 同一订单成为多条点评的高分首选 |
| 详情复核 | 0或-5 | 已复核0；未复核-5并禁止高置信 |

## 硬边界

- `confirmed`只由强订单标识产生。
- `high_confidence`除总分至少75外，还必须满足门店映射已复核、房型至少25分、日期大于0且无硬冲突、有效入住状态、金额大于0、详情已复核、唯一且无重复命中。
- 前两名分差小于10、时间硬冲突、房型0分或首选订单跨点评重复时为 `ambiguous`。
- 取消、关闭、NoShow、0元和未复核订单保留在前5名候选中，并写出扣分原因。
- 非携程渠道订单直接排除。
- `not_found`必须带上已搜索窗口和跳过/失败原因。

## 隐私与范围

- 评分输出不包含客人真实姓名、手机号、UID、头像或IM members。
- 即使订单详情有客人信息，也不得用于反推匿名点评者身份。
- 结果属于携程OTA渠道证据，不代表PMS全渠道或全酒店经营真相。
