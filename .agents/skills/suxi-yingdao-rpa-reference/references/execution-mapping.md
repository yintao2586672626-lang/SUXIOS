## 直接可参考

| 影刀分类 | 宿析优先执行面 | 必须验证 |
| --- | --- | --- |
| 影刀概述、快速入门、功能文档 | 项目现有入口、界面和任务流程 | 公开操作步骤、当前版本差异、宿析是否已有对应能力 |
| 条件判断、循环、等待 | 项目现有 JavaScript、PHP 服务与任务状态机 | 终止条件、超时、重试上限、失败回显 |
| 相似元素、网页自动化 | Browser/Chrome、结构化响应、必要时 DOM | 页面授权、唯一定位、分页终止、缺失状态 |
| 数据表格、数据处理 | 现有导入服务、Node/PHP 纯函数 | 字段语义、类型、空值、来源和回读 |
| Excel/WPS | Spreadsheets 或项目导入导出能力 | 工作簿、工作表、公式、日期和旧数据兼容 |
| 流程/应用 | 项目现有流程、任务和服务入口 | 状态变化、副作用、幂等和恢复 |
| 人工智能AI | 现有 AI 服务或明确授权的模型工具 | 输入范围、模型输出、失败状态和事实边界 |
| 网络 | 项目现有 HTTP/API 客户端 | 鉴权、分页、限流、响应字段和错误语义 |
| 接口文档、开放API | 项目现有服务或明确授权的外部 API 客户端 | 接口版本、鉴权、字段、分页、限流、幂等和错误码 |
| 常见问题 | 复现、日志、聚焦诊断和回归样例 | 当前症状是否一致；帮助文章只能作为线索，不能替代根因证据 |
| 专题文档、解决方案 | 项目现有业务流程和能力组合 | 输入输出、适用条件、失败状态及宿析独立实现证据 |

## 需明确授权或环境

| 影刀分类 | 条件 |
| --- | --- |
| 桌面软件自动化、鼠标键盘、对话框、操作系统 | 用户明确要求操作本机应用，并且 Computer Use 或等价工具可用 |
| 工作队列 | 目标队列、并发、重试、幂等和可观察状态已经定义 |
| 外部平台写入 | 用户明确授权具体平台、账号、目标和写入范围 |
| Excel/WPS 活跃窗口操作 | 用户明确选择工作簿，且不会覆盖未保存改动 |
| 管理文档中的组织、成员、权限和任务写入 | 目标账号、组织、权限范围和副作用已经明确授权 |

## 默认不支持

- 手机操作自动化：除非用户提供可控设备、授权范围和验证方式。
- 影刀自定义指令运行时：宿析不会假装拥有影刀客户端、插件或其私有运行环境。
- 影刀专有 UI、隐藏后台和内部算法：只复用公开可见的任务合同，不复制外观或臆造实现。

## 落地合同

```text
影刀公开方法 -> 宿析指定入口 -> 独立实现动作 ->
可观察结果 -> 保存与回读（如适用） ->
超时/缺失/权限/终止失败 -> 聚焦验证
```

文档内容只作为方法证据；当前环境能否执行必须由宿析自己的运行证据证明。
