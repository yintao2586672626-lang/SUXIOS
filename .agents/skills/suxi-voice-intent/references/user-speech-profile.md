# 用户口语画像（初始版）

## 证据边界

- 来源：当前会话中用户主动提交的两段普通话语音转录文字。
- 未获取、未保存原始音频，因此不能判断具体声学口音、音高、语速或发音误差。
- 以下只是帮助理解的初始模式，不是人格判断；用户后续明确纠正时立即覆盖。

## 已观察到的表达模式

- 使用自然、连续的普通话口述，常先说明背景，再在后半段提出具体要求。
- 会出现“嗯、对吧、反正、就是说、然后”等连接词，以及少量重复和边说边修正。
- 常用“这个、它、你给我的这个”等近距离指代，需要结合最近任务对象解析。
- 需求通常是结果导向：希望助手自行判断技术细节、寻找 Skill/插件或现有能力，不希望为小错字反复确认。
- 酒店业务和写系统是稳定背景，出现疑似专业词时应优先结合宿析OS项目语境判断。

## 默认理解策略

1. 去掉不改变意义的连接词和重复。
2. 找到最后一个清晰的目标或动作要求，将前文视为背景或约束。
3. 优先恢复业务对象、动作、范围和验收结果，不逐字复刻口语顺序。
4. 对普通同音字直接按上下文纠正；对否定、数字、平台、门店、文件和真实数据动作保持谨慎。
5. 意图复杂或高风险时，用一句话复述核心目标；简单任务直接执行。

## 当前不应推断的内容

- 不因用户来自陕西就预设具体方言替换规则。
- 不把“对吧”自动当成用户已经批准某项高风险操作。
- 不把“你去录一下”扩展成后台持续录音许可；当前只记录抽象语言模式。
