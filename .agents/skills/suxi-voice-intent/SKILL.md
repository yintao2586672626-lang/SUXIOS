# 宿析OS语音意图理解

## 按需自动触发

- 项目规则先对每条消息做字面一致性判断；句子连贯、目标明确、关键字段无冲突时不加载本 Skill。
- 出现疑似转录噪声、酒店同音词、自我纠正、断裂语序、指代不清，或用户明确要求理解语音时自动触发；用户无需点名，普通对话中也不提示 Skill 名称。
- 本 Skill 是唯一自动语音入口。只有出现疑似转录错误或真实歧义时，才按需读取口语画像、酒店热词或 `~/.transcript-fixer/contexts/suxios-hotel.md`；不自动并行调用全局 `$transcript-fixer` 或其他语音插件。
- 单条聊天消息只走内存快速路径：不创建文件、不运行 CLI、不写 SQLite、不调用外部 API。
- 用户提供整份转录文件、要求批量处理或明确要求持久学习时，才进入下方的 `transcript-fixer` 文件工作流。
- 用户明确说明“这是键入原文”“逐字保留”或提供需精确复制的代码、路径、ID、引文时，保留原文优先于自动纠错。

## 处理流程

1. 先做字面一致性检查：如果原句已经能唯一确定目标和范围，保持原意直接执行。
2. 只有出现断裂语序、明显同音词、连续口语、自我纠正或上下文对象不唯一时，才把输入视为可能存在转录噪声；口音本身不是错误。
3. 需要用户表达习惯时读取 [用户口语画像](references/user-speech-profile.md)；需要酒店术语或同音判断时读取 [酒店热词表](references/hotel-hotwords.md)。不要为了每条消息固定加载二者。
4. 静默清理不承载意图的口头语、重复和已被后文覆盖的自我修正，保留否定词、范围词、动作词、数字、日期、平台、门店与文件名。
5. 结合当前任务、最近一轮上下文和项目证据排列候选解释，不要只按单个同音词判断，也不要用旧任务补全当前缺失对象。
6. 将理解结果内部整理为：目标、涉及范围、明确不做、验证方式。意图清楚时直接执行，不要求用户改成技术语言。
7. 只有关键歧义会改变操作结果、业务口径或数据安全时，才用一句话确认；其余高置信度小错误静默纠正并继续。

## 上下文绑定与置信度

按以下顺序绑定“这个、那个、它、当前门店”等省略对象：

1. 当前句中最后一个明确对象。
2. 已验证的当前页面、当前选择门店或本轮正在处理的任务对象。
3. 同一任务最近一轮中唯一且未被切换的对象。

不要从无关旧任务、历史门店或未验证页面状态中补对象。内部使用三级置信度：

- 高：只有一个上下文一致的解释，静默纠正并执行。
- 中：存在轻微文字歧义但不改变关键结果，采用最合理解释，并在结果中简短说明假设。
- 低：存在两个以上合理解释，或歧义涉及关键动作、数值、门店、平台、数据事实或发布，先问一个决定性问题。

## 关键歧义门槛

以下内容不确定时必须确认：

- “要/不要”“改/不改”“删除/保留”“发布/只在本地”等相反动作。
- 数字、日期、金额、比例、端口、门店、平台酒店 ID、系统酒店 ID。
- 文件路径、命令、账号、Cookie、令牌、真实数据写入和不可逆操作。
- OTA 渠道口径与全酒店经营口径。

确认时只指出决定执行结果的那个词，不要求用户重新说完整段落。

## 解释规则

- “对吧、嗯、然后、就是说、反正”等通常作为口语连接词，不自动解释为确认、授权或范围扩大。
- 后一句明确纠正前一句时，以后一句为准。
- “这个、那个、它”优先指向最近的明确对象；存在两个同等可能对象且会改变执行结果时再确认。
- 酒店/OTA语境中的候选词优先查热词表；代码语境仍保留真实技术含义，例如“协程”不能一律改成“携程”。
- 用户说“看一下、优化、处理一下”时，只在当前明确区域内推进，不扩成全项目审计。
- 继续遵守项目事实边界：OTA 数据不自动代表全酒店数据，模拟、未验证和采集失败不能被说成真实成功。

## 与 transcript-fixer 协作

- 单条聊天语音转录仍由本 Skill 直接结合上下文理解，不为了偶发错字写入永久词典。
- 用户提供转录文件、要求批量纠错，或明确要求“以后记住这个纠正”时，调用全局 `$transcript-fixer`，领域固定为 `suxios-hotel`。
- 默认只调用 `C:/Users/Administrator/.codex/skills/transcript-fixer/scripts/run_safe_stage1.ps1`；它固定本地 Stage 1 安全模式并为每次处理创建新的输出目录。不要使用默认 Stage 3，不要使用 `--apply-all`，也不要在原文件旁重复运行，以免触发外部 API 或自动回写原文件。
- Native AI 纠错由当前 Codex 结合 `~/.transcript-fixer/contexts/suxios-hotel.md` 完成；除非用户另行明确授权，不配置 GLM、Anthropic 或其他外部 API，不上传转录文字。
- 只有确定、可重复且不含敏感信息的映射才进入 `suxios-hotel` 词典；常用词同音歧义只作为带条件的上下文提示，不做机械替换。

## 学习与隐私边界

- 当前能力作用于转录后的文字，不声称能够访问麦克风、录制原始音频、训练声纹或修改 Codex 上游转录模型。
- 不保存原始音频或整段私人转录。只保留完成理解所需的抽象表达习惯和非敏感专业词。
- 用户明确说“记住这个纠正”“以后都按这个理解”时，才把稳定映射补入对应 reference；一次性的错字不自动永久化。
- 不把账号、密码、Cookie、令牌、手机号或个人敏感信息写入画像和热词表。
- 用户的明确纠正永远高于画像、热词表和上下文推断。

## 最小回归样例

- “协程昨天间页数据”：酒店 OTA 语境可还原为“携程昨天间夜数据”，但执行查询前仍需有明确门店。
- “把美团入驻率改成八十五再发布”：可还原“入住率 85%”，但必须先确认这是运营目标还是企图修改真实采集事实。
- “熟悉OS那个数据页”：可还原“宿析OS数据页”；当前页面不可验证且没有唯一对象时，只确认具体页面。
