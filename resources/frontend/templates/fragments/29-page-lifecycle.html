1

筹建阶段

从"拍脑袋想投"到"算清再投"

智略 + 智算 + 智策

阶段定位：用"数据算账"替代"经验拍板"，让酒店投资的"第一笔钱"花在刀刃上。

宿析解法：智略（战略推演引擎）+ 智算（量化风控模型）双轮驱动

把"模糊的投资冲动"变成"可落地的数据蓝图"，让筹建期的每一个选择都有"数"可依。

核心问题：我的钱够不够？投在哪里能赚回来？
1. 智略定方向

用"战略推演"画清"投哪里、投什么"

示例测算 IRR 与模式对比须根据项目现金流、投资额和退出假设计算；当前示意不代表真实项目结果
盈亏平衡点 根据总投资、稳定期现金流和爬坡周期测算，不预置回本时间
区域适配性 根据可核验客流、需求与竞争数据判断，不预设“高潜力”结论
2. 智算控风险

用"量化拆解"算清"钱够不够、怎么花更赚"

示例约束 装修投入比例与 RevPAR 不存在固定提升关系，须结合房价、出租率和改造周期测算
资金规划 根据收入、成本、付款节点和营运资金输入测算现金流缺口；未录入时保持缺失
示例风险 竞对密度只是投资风险输入之一，不能单独推导 IRR
3. 决策依据

按已录入且可核验的项目、市场与财务信息形成辅助判断

区域客流增速 待接入来源
竞对密度 待按选定范围统计
案例参考（仅在接入可核验同区位样本后展示）
使用边界
筹建结论来自用户录入参数、已接入数据和明确假设；系统展示测算依据与风险，不替代财务、法律、消防或工程尽调。
2

开业阶段

从"手忙脚乱上线"到"开业即高效"

项目 + 清单 + 验收

阶段定位：用项目、任务、责任人、截止时间和验收记录管理开业准备；系统建议用于提示风险，不代表事项已经完成。

宿析解法：开业项目 + 标准检查清单 + 状态跟踪 + 建议与验收记录

开业进度以保存的任务状态和验收证据为准；没有完成记录时，不推断人员就绪、系统联通或经营效果。

核心问题：开业事项是否有人负责、按期完成并通过验收？
1. 系统与数据就绪

按配置、数据接入和验收记录核对上线准备

上线时效 按实际配置、数据接入和验收记录统计，不预设提效比例
开业就绪度 分别核验 PMS、OTA、支付、证照、安全、房型房价库存和人员培训；未提供验收证据的项目保持待确认
2. 数据不孤岛

只汇总已授权、已接入且来源明确的数据

单客视图 仅展示已授权、已接入的客户偏好、消费与服务记录，不承诺固定提效
连通状态 以接口监控和同步日志为准，不预设延迟或可用率
3. 员工速上手

用任务、培训和实操验收记录跟踪人员准备

培训时间 由门店按岗位、课程和实操验收记录统计
操作熟练度 以实际考核和操作日志衡量，不预置失误率
使用边界
开业准备会汇总系统配置、数据接入、人员培训和验收状态；实际进度与经营效果以项目记录为准，不承诺固定工期或盈利提升。
3

运营阶段

从"凭感觉调策略"到"数据给答案"

数据核验 → 建议确认 → 执行留痕 → 效果复盘

阶段定位：先核对数据来源与口径，再形成原因假设、待确认建议、人工审批动作和效果复盘。

宿析解法：数据核验 → 原因分析 → 风险提示 → 策略模拟 → 执行留痕

规则或 AI 输出只作为解释和建议；没有审批、执行证据和同口径后续数据时，不宣称动作已执行或效果已实现。

核心问题：已核验指标为什么变化，下一步应复核或执行什么？
1. 策源找病灶

按来源和口径汇总可用指标，先确认“发生了什么”

示例输入 市场仪表盘按已核验数据展示指标变化；未接入时不生成变化值
数据覆盖 按已接入来源展示 OTA 渠道与会员指标；缺失项保持缺口
2. 策析挖根源

把可能原因列为待验证假设，不把相关性写成确定因果

原因假设 竞品分流、会展变化和客群变化需分别取证，不能用无来源比例拼成确定根因
影响量化 仅在具备对照数据与可解释方法时估算各因素贡献
3. 策见早预警

基于已接入数据生成风险提示，缺来源时不生成结论

价格预警 只有采集到竞品价格变化及对应日期后才展示
事件预警 外部事件与客流影响需标明来源、时间和不确定性
4. 策案给选项

用输入假设做情景模拟，提供人工复核选项

模拟结果 调价与套餐方案只输出基于输入假设的情景值，不承诺实际 RevPAR 提升
选项说明 输出"保守方案""进取方案""平衡方案"各方案适用场景
5. 策行追效果

通过审批、执行证据和复盘记录核对动作效果

执行效率 按审批、执行和回执时间记录，不预置提效结果
效果追踪 执行后按同口径数据记录销量与出租率变化；没有执行证据时保持待观察
使用边界
OTA 数据只支持对应渠道的经营分析；全酒店出租率、ADR、RevPAR 或总收入必须另有 PMS、CRS 或全渠道证据。建议需人工确认，效果需用执行后的同口径数据复盘。
4

扩张阶段

从经验复制到扩张条件评估

智投 + 智瞰 + 智联
核心问题：新市场能不能进？复制哪家店的模式？
阶段定位

把市场输入、标杆样本、协同记录和财务假设拆开呈现，用于辅助判断是否继续尽调；不承诺市场进入或复制结果。

宿析解法

市场评估 + 标杆比较 + 协同效率记录

结果取决于输入范围、样本可比性和数据来源；缺少区域、竞对、消费或成本证据时只显示待补项。

1. 智投评市场

按已录入且来源明确的市场指标生成评估草案

区域客流、竞对、消费和政策等输入需分别标明地区、周期与来源；规则或模型输出均需人工复核

客流增速：待接入指定区域与周期的数据
竞对密度：待确认范围与竞对名单后统计
消费能力：待标明统计来源与年份
综合结论：基于已保存假设测算 ROI，并同步展示口径与风险
2. 智瞰选标杆

用同周期、同来源、同口径数据比较候选门店

按已接入、同口径数据查看各门店 OCC、ADR 与 RevPAR；标杆结果仅作比较参考，不自动归因为“成功模式”

标杆识别：仅在同口径套餐收入与总收入均可核验时计算贡献率
模式对比：按同周期、同会员口径比较复购率
风险提示：客源集中度与 RevPAR 波动需分别取证后再判断关联
3. 智联提效率

记录总部与门店协同事项，并用前后同口径数据复盘

采购、营销或会员数据只有在已授权接入并形成任务记录后才参与比较；当前页面不代表自动调度或降本结果

成本变化：按同口径执行前后成本核算
效率变化：按采购任务实际周期统计
会员共享：按授权范围和同口径复购数据评估
4. 决策依据：扩张决策数据看板
智投：市场评估雷达图
客流、竞对密度与消费能力均需标明地区、周期和来源
智瞰：标杆门店对比表
示例字段：套餐收入贡献、复购率；无可核验数据时显示缺口
智联：成本对比图
展示同口径成本变化与采购周期，不使用演示数字代替
使用边界
扩张评估是尽调辅助，不是进入市场、复制门店或预期收益的保证。所有结论应同时展示事实来源、人工假设、缺失数据和风险。
5

转让阶段

从"纠结何时卖"到"算清变现价"

智算 + 智略
核心问题：现在卖划算吗？能卖多少钱？
阶段定位

把经营输入、估值假设、时机信号和风险拆开呈现，用于比较转让情景；不承诺成交价格、成交时间或资产收益。

宿析解法

估值情景测算 + 转让时机评分 + 风险复核

系统只基于已录入参数和现有记录输出辅助区间；租约、证照、债务、税务、法律和真实成交样本仍需线下尽调。

1. 智算定价格

按人工录入的经营与资产参数计算估值情景

当前结果是系统测算区间，不等同于具备完整市场法、收益法与成本法证据的正式资产评估报告

市场供需：仅在来源、地区、周期和样本范围可核验时使用
折旧成本：按资产类别、年限与会计口径输入
未来收益折现：展示现金流、折现率与终值假设
可比案例：只引用来源、日期和口径可核验的成交样本
2. 智略择时机

按趋势、评分、节假日窗口和数据质量生成时机辅助评分

评分是规则化比较结果，不证明未来成交条件；再投资收益须使用新项目现金流另行测算

时机对比：按不同退出日期测算售价与持有现金流情景
再投资收益：按新项目现金流重新测算 IRR，不沿用转让项目结果
风险预警：竞品入市时间和售价影响须有来源并按情景表达
3. 决策依据：转让决策数据看板
智算：资产估值报告摘要
展示估值区间、折旧口径与收益折现结果及其假设
智略：时机收益对比表
示例字段：目标退出时间、售价情景与再投资 IRR；结果取决于输入假设
风险提示
竞品入市与政策变化按来源和情景范围提示，不给出无依据幅度
使用边界
转让决策辅助会拆分事实、假设、测算与风险：资产定价提供售价区间，时机推演比较不同退出情景。结果用于辅助人工决策，不承诺成交价格、退出时点或资产收益最大化。
